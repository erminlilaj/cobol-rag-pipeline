## Important Model Setting

Direct `final_scripts` answers must not be rewritten by Granite by default.

Current setting in `config/default.yaml`:

```yaml
answers:
  llm_polish_final_scripts: false
```

Why:

- The bad combined answers came from Granite inventing generic business stories such as customer/travel/booking.
- Deterministic final_scripts answers should stay exact and citation-based.
- Retrieval and LLM should only help when direct evidence cannot answer.

If hallucinations return, check:

```bash
echo "$COBOL_RAG_LLM_POLISH_FINAL_SCRIPTS"
```

It should be unset or false:

```bash
unset COBOL_RAG_LLM_POLISH_FINAL_SCRIPTS
```

Also restart `uvicorn`; an old server can keep old behavior alive.

## Current RAG Improvements

The RAG now has deterministic routing for these PDCBVC questions:

- Copybooks and their roles.
- `TWCOB-FASE` phase dispatch to `BROWSE-FASE1`, `BROWSE-FASE2`, or `ABEND00`.
- `TWCOB-FUNZIONE in I/A/C/D/P` with `TWCOB-ID-SISTEMA = 'IP'`.
- Semaphore `PDAGGVIP` and `PXCSEMAF` request preparation.
- `PD1VOCI` COMMAREA preparation in `INIZ-PARAM`, not `LINK-PD1VOCI`.
- `PDRTWA2`/`TWCOB-*` fields moved into `PD1VOCI`.
- `TWCOB-VARCONT-NUMFUNZ` mappings for `PD1VOCI-FUNZIONE` and `PD1VOCI-TIPO-VOCE`.
- `PD1FS00` call and returned session fields.
- `PD1FS00-SESS-FLAG` display values:
  - `1` -> `Aperta`
  - `2` -> `Chiusa`
  - `3` -> `Liquidata`
- Page total calculation in `CALCOLA-NPAG`.
- `PREP-RIGA` row construction.
- Error paths with real paths separated from message/cursor field evidence.
- Unknown program/file/variable guards.

## Bad Answers That Were Fixed

The combined test previously produced bad answers like:

- Copybooks described as customer/travel/booking data.
- `TWCOB-FUNZIONE + IP` described as login/menu/flight/hotel flow.
- `PD1VOCI` fields said to be prepared in `LINK-PD1VOCI`.
- Wrong `TWCOB-VARCONT-NUMFUNZ` mappings.
- Page count hallucinated as `(TWCOB-VARCONT-NUMFUNZ + 2) * 5`.
- Missing session flag mapping for `Aperta`, `Chiusa`, `Liquidata`.

Root cause:

- Some question variants missed deterministic routing.
- `llm_polish_final_scripts` was enabled by default, so Granite could rewrite evidence and add fake details.

Fix:

- Added direct evidence handlers.
- Added gold tests for the exact bad questions.
- Disabled final_scripts LLM polishing by default.
